- **Predictable**: Same layout on every screen size
- **Natural reading order**: Top to bottom, like a document
- **Scannable**: Easy to find sections by scrolling
- **Mobile-friendly**: Works perfectly on narrow screens
- **Simple to implement**: Just change CSS, no structural changes
- **Respects importance**: First sections = most important

### Cons
- **Vertical scrolling**: May require more scrolling on tall forms
- **Wasted space**: On wide screens, lots of whitespace on sides
- **Dense feeling**: Many collapsed sections may look intimidating

.form-panels-container {
  display: flex;
  flex-direction: column;
  gap: 16px;
  max-width: 800px;

## Recommendation

### For Immediate Implementation: **Proposal A (Single Column)** or **Proposal B (Two-Column Fixed)**

**Why:**
1. Both are CSS-only changes (no structural modifications)
2. Both eliminate the chaos of auto-fit
3. Both provide predictable, scannable layouts
4. Can be shipped today with minimal risk

### Suggested Approach:

```css
/* Default: Single column for clean vertical flow */
.form-panels-container {
  display: flex;
  flex-direction: column;
  gap: 16px;
  max-width: 900px;
  margin: 0 auto;
  padding: 20px;
}

/* Optional: Two columns on very wide screens */
@media (min-width: 1400px) {
  .form-panels-container {
    display: grid;
    grid-template-columns: repeat(2, 1fr);
    max-width: 1400px;
  }
}
```

### Additional Quick Wins (No CodeGen Changes):

1. **Collapse technical sections by default** - Embedding, System Metadata
2. **Add "Show Technical Details" toggle** - Lets users opt-in to technical noise
3. **Limit max-height on text fields** - Prevent embedding vectors from dominating
