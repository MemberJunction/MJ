# Pinned dependencies for the sqlglot-ts Python microservice.
#
# Why pin?
#   sqlglot's behavior changes across releases — new dialect quirks, regression
#   fixes, and parser improvements can subtly alter conversion output.
#   Pinning gives us reproducible conversion results and a controlled upgrade
#   surface (when we bump the pin, we re-run the full v5 migration regression).
#
# Upgrade procedure:
#   1. Bump the pin below.
#   2. Run `mj sql-audit` and the SQLConverter test suite.
#   3. Regenerate `migrations-pg/v5/` via `mj migrate convert`.
#   4. Apply migrations against a fresh PG database via `mj migrate`.
#   5. Diff the regenerated PG migrations against the prior set — investigate any
#      unexpected changes before merging.

pydantic~=2.12.0     # Request/response models (FastAPI dep, pinned for clarity)
