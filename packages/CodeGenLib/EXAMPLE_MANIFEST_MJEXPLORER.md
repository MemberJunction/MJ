# Example Manifest: MJExplorer (Client-Side Angular)

This is an example of the manifest generated by `mj codegen manifest` when run against the **MJExplorer** Angular application.

- **Dependencies walked:** 1,179
- **Packages with @RegisterClass:** 17
- **Total classes discovered:** 720

The manifest uses **named imports** for every `@RegisterClass` decorated class and places them all in an exported `CLASS_REGISTRATIONS` array, creating a static code path that prevents tree-shaking from removing them.

Notice the client-specific packages: Angular components (`ng-artifacts`, `ng-ask-skip`, `ng-auth-services`, `ng-core-entity-forms`, `ng-dashboards`, `ng-explorer-core`, `ng-explorer-settings`, `ng-file-storage`, `ng-react`). Server-only packages like AI providers, encryption, and scheduling are absent.
