MJ Support Widget — guest test harness

A bare third-party page. Serve it on http://localhost:8080 (a whitelisted origin) so the embed is a true cross-origin drop-in.

One widget per page. The conversations runtime is a process-wide singleton, so two widgets on the same page would share one agent-dispatch context and collide. That mirrors real usage (a customer embeds one widget per site).

To demo cross-guest isolation: open this page in two separate browser windows (or one normal + one incognito). Each window is its own JS context → its own guest session with a distinct ExternalID → fully isolated. Server-side RLS enforces the isolation regardless; the two windows just let you see it.
  1. Shadow-DOM isolation: the loud yellow/purple/hotpink page must NOT bleed into the widget, and the widget must not restyle this page.
  2. Guest text turn (no login): open the launcher, type a message — it reaches the pinned Sage agent with no MJ login.
  3. Voice: click the mic, talk. The voice ceiling is enforced server-side at the realtime mint.
  4. Cross-guest isolation: open a second window (incognito), chat there too — each window's conversation is private to that guest.
  5. Magic-link upgrade: trigger upgrade from a guest session and complete the link — the session escalates to a resolved account, keeping the conversation.
