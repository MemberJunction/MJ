## 2024-05-19 - Cache Initialization and Duplicate keys
**Learning:** `BaseEntity.GetFieldByName` lookups can become a bottleneck when repeatedly iterating an entity's fields (especially during serialization and `SetMany`). Attempting to memoize field names using the same cache dictionary as `CodeName` causes a collision hazard and incorrectly overrides names. Additionally, TypeScript's strict definitions and `Array.find` semantics require carefully handling duplicate insertions into Maps to ensure the earliest element encountered persists (using `if (!map.has(...)) map.set(...)`).
**Action:** When implementing caching in heavily iterated paths, strictly separate caches based on the exact property being queried (e.g., separate caches for `Name` and `CodeName`). Replicate the semantics of previous native array functions explicitly when constructing caching objects, especially for edge cases like duplicate elements.
