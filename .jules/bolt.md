## 2024-05-24 - O(1) Hash Map Optimization in Core Lookups\n**Learning:** The `Metadata` and `BaseEntity` classes were performing O(N) array scans for frequently queried items (`EntityByName`, `EntityByID`, `GetFieldByName`). For `Metadata`, delegating to the provider is preferred, but returning `undefined` unconditionally without populating a fallback Hash Map ensures safe misses while avoiding expensive array scans.\n**Action:** Use `Map` structures for fast, frequent lookups and ensure fallback paths maintain functional correctness without inadvertently degrading performance.
