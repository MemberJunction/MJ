AI & Agents
What it does

One platform. Database to AI agents.

Define your schema. MemberJunction generates everything that depends on it — and provides the runtime that uses it.

Metadata-driven

Your database schema is the source of truth. Entities, fields, validation, audit trails, and permissions are all metadata — never hand-written boilerplate.

AI-native

26 providers behind a single abstraction. Anthropic, OpenAI, Gemini, Bedrock, Mistral, Groq, and more — swap models without touching application code.

Agent Framework

Build autonomous workflows with Loop and Flow agents. First-class MCP & A2A support means your agents speak the protocols Claude, Cursor, and others already use.

CodeGen

One command generates strongly-typed entity classes, GraphQL schemas, Angular forms, stored procedures, and views. From schema change to production-ready code in seconds.

Angular UI

100+ reusable components, auto-generated CRUD forms, a dashboard framework, and a design token system that supports white-labeling out of the box.

Actions Framework

A pluggable interface for exposing functionality to agents, workflows, and low-code builders. Discoverable, declarative, type-safe.

In code

What working with MJ looks like

A typical pattern: load entities, work with strongly-typed properties, save. The metadata system handles validation, audit, FK relationships, and AI capabilities behind the scenes.

import { Metadata, RunView } from '@memberjunction/core';
import { UserEntity } from 'mj_generatedentities';

// Load a strongly-typed entity — types are generated from your schema
const md = new Metadata();
const user = await md.GetEntityObject<UserEntity>('Users');
await user.Load(userId);

// Mutate via real properties — no .Get('FieldName') stringly-typed access
user.Status = 'Active';
user.LastLoginAt = new Date();

// Save — validation, audit (Record Changes), and pub/sub fire automatically
const saved = await user.Save();

// Query collections with RunView — batched, cached, multi-server aware
const rv = new RunView();
const result = await rv.RunView<UserEntity>({
    EntityName: 'Users',
    ExtraFilter: `Status='Active'`,
    OrderBy: 'LastLoginAt DESC',
    MaxRows: 50,
    ResultType: 'entity_object'
});

Same pattern from the API server, an Angular component, a Node script, or an AI agent action — one metadata layer underneath all of it.
