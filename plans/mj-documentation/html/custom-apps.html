Building custom apps on MJ

MJ Explorer is the reference app — but it's also a template. You can ship your own branded application using the same dashboard framework, the same auto-generated forms, the same lazy-loading infrastructure. This page is the entry point.

Overview

A "custom app" on MJ is an Angular application that:

  • Bootstraps with @memberjunction/ng-base-application
  • Pulls its data via @memberjunction/graphql-dataprovider from your MJAPI
  • Defines its own navigation, dashboards, and custom forms
  • Inherits MJ's authentication, theming, and component library

Scaffold a custom app

mj init my-app --template custom-app
cd my-app
npm install

You get a minimal Angular app preconfigured with MJ's auth, design tokens, and a starter dashboard:

my-app/
├── src/
│   ├── app/
│   │   ├── dashboards/         # Your custom dashboards
│   │   ├── custom-forms/       # Your custom entity form overrides
│   │   ├── app.module.ts
│   │   └── app.routes.ts
│   ├── styles.scss              # Brand theme overrides
│   └── main.ts
├── metadata/                   # App-specific metadata (Apps, nav items, etc.)
└── mj.config.cjs

Building dashboards

Dashboards extend BaseDashboard from @memberjunction/ng-dashboards. They get automatic page chrome, loading state, user preference persistence, and lazy-loading wiring.

import { BaseDashboard, RegisterClass } from '@memberjunction/ng-dashboards';

@RegisterClass(BaseResourceComponent, 'SalesPipelineDashboard')
@Component({
    selector: 'app-sales-pipeline',
    templateUrl: './sales-pipeline.component.html'
})
export class SalesPipelineDashboard extends BaseDashboard {
    public async loadData() {
        // Read entities, build state
    }
}

Read the full Dashboard Best Practices guide for state management patterns, the engine class approach, and the page chrome (<mj-page-layout> + header/body trio).

Custom entity forms

Every entity gets an auto-generated form. To customize one, extend the generated form class — this preserves all the wiring while letting you change layout, add tabs, or override behavior:

import { UserFormComponent } from '../../generated/Entities/User/user.form.component';

@RegisterClass(BaseFormComponent, 'Users')
@Component({ /* ... */ })
export class UserFormComponentExtended extends UserFormComponent {
    // Override anything, add new tabs, wire up custom panels
}

Lazy-loaded modules

MJ Explorer and custom apps use code-split lazy loading. To add a new feature module that loads on demand:

  1. Mark your package's package.json with subpath exports so it can be lazy-imported.
  2. Decorate your component with @RegisterClass(BaseResourceComponent, 'YourDriverClass').
  3. Run mj codegen manifest --lazy-config — the manifest updates automatically.

See the Lazy Loading guide for the full pattern.

Theming & white-label

MJ uses a CSS custom property design token system. Override any token in your app's styles.scss and you've reskinned the entire UI:

/* styles.scss */
:root {
    --mj-brand-primary: #0066ff;
    --mj-brand-primary-hover: #0052cc;
    --mj-text-primary: #0c1116;
}

[data-theme="dark"] {
    --mj-bg-page: #0a0c10;
    --mj-bg-surface: #11141a;
}

See the Theming guide and Design Token guide for the full token list and dark-mode patterns.

User preferences

Persist per-user UI state via UserInfoEngine (NEVER localStorage — preferences must follow the user across browsers and devices):

import { UserInfoEngine } from '@memberjunction/core-entities';

// Read (sync, cached at bootstrap)
const raw = UserInfoEngine.Instance.GetSetting('myApp.sidebarCollapsed');

// Write (debounced, fire-and-forget)
UserInfoEngine.Instance.SetSettingDebounced('myApp.sidebarCollapsed', 'true');

Custom actions

Actions are MJ's pluggable interface for exposing functionality to agents, workflows, and low-code builders. Define an action once and it becomes invokable from the UI, from agents, from the Actions Framework, and from external systems via MCP.

See the Actions documentation for action authoring patterns.

Deployment

MJ apps deploy as standard static Angular bundles + a Node MJAPI service. Both run cleanly in Docker. See the Ecosystem page for Docker images and example compose files.
