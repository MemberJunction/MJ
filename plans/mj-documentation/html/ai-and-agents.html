AI & Agents

MJ's AI stack covers everything from raw provider abstraction up to autonomous agent workflows. 26 providers, a unified prompt engine, the agent framework, native MCP & A2A support, and a published catalog of Claude Code skills for developers using MJ in their own projects.

Overview

The AI layer is split into five concerns, each with its own package or set of packages:

  • Providers — adapters for each LLM / embedding / voice / image vendor. Uniform interface.
  • AI Engine — model routing, caching, cost tracking, retries.
  • Agent Framework — Loop & Flow agents, sub-agent composition, conductor orchestration.
  • Prompt Engine — database-managed prompt templates, runners, context injection.
  • Vectors / Knowledge — embedding storage, similarity search, autotagging, RAG+.

26 AI Providers

Every provider implements the same BaseLLM interface (or BaseEmbeddings, BaseImageGenerator, etc.). Application code never imports a provider directly — it asks the AI Engine for a model by name and gets the right driver back.

AI Engine

The AIEngine handles model selection, vendor routing, cost tracking, response caching, and retries. You configure a model in metadata (cost, capabilities, max tokens) and the engine matches your request to the cheapest model that meets your requirements:

import { AIEngine } from '@memberjunction/aiengine';

const result = await AIEngine.Instance.RunPrompt({
    SystemPrompt: 'Classify this support ticket.',
    UserMessage: ticket.Body,
    ModelType: 'classification',    // engine picks an appropriate model
    MinPowerRank: 2,                  // require at least this much capability
    ResponseFormat: 'JSON',
    Temperature: 0
});

Agent Framework

MJ ships two agent execution strategies — both extend BaseAgent:

Loop Agents

The classic ReAct-style agent: think → call a tool → observe → repeat. Best for open-ended tasks where the next step depends on what came back from the last one.

Flow Agents

Deterministic, node-based workflows you build in the Flow Editor. Best for processes where the steps and branches are known in advance.

Both agent types can:

  • Invoke any registered Action via the Actions Framework.
  • Spawn sub-agents (conductor pattern).
  • Stream output progressively to the client.
  • Pause for human approval via the agent request queue.
  • Be triggered manually, on a schedule, or by entity events.
import { BaseAgent, RegisterClass } from '@memberjunction/ai-agents';

@RegisterClass(BaseAgent, 'TicketTriage')
export class TicketTriageAgent extends BaseAgent {
    protected async runStep(ctx: AgentContext) {
        // loop logic, action invocations, sub-agent calls...
    }
}

Prompt Engine

Prompts live in the database — versioned, organization-scoped, and composable. The AIPromptRunner handles template rendering, context injection, model selection, and result parsing in one call:

const params = new AIPromptParams();
params.prompt = await promptEngine.GetPrompt('Summarize Document');
params.data = { document: doc.Body, audience: 'executive' };

const result = await new AIPromptRunner().ExecutePrompt(params);

Vectors & RAG+

Every entity can have its content embedded into vectors using the EmbedTextLocal pattern from the BaseEntity Server-Side Patterns guide. The vector store lives in your database (no separate vector DB to operate). Semantic search and RAG+ are first-class — see the Search Scopes & RAG+ guide.

MCP Server

MJ exposes its entities, actions, queries, and prompts as MCP tools. Connect Claude Desktop or Cursor to your MJ instance and your agents can read and write entities, run actions, and execute queries — all governed by your MJ permission model.

// In Claude Desktop's claude_desktop_config.json
{
    "mcpServers": {
        "memberjunction": {
            "command": "npx",
            "args": ["-y", "@memberjunction/ai-mcp-server"],
            "env": { "MJ_API_URL": "https://your-mj.example.com/graphql" }
        }
    }
}

A2A Server

MJ also speaks the Agent-to-Agent protocol, letting your agents be invoked by external systems (or other MJ deployments) over a standardized HTTP/JSON contract. Pair with the MCP server when you want both protocols on the same backend.

Claude Code Skills Catalog

If you're using Claude Code to develop on MJ, the repo ships a growing set of skills — task-specific playbooks Claude can invoke. Install one into your project and Claude gains that capability for any subsequent task.

Skills are auto-discovered
