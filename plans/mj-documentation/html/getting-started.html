Getting Started

From nothing to a running MemberJunction API + Explorer in under 10 minutes. Then create your first entity and see auto-generated forms, GraphQL, and typed TypeScript classes appear.

Rendered from MJ/README.md · DEPLOYMENT.md

Overview

MemberJunction installs as an npm package + a database. You scaffold a project with the CLI, point it at SQL Server or PostgreSQL, run the bootstrap migrations, and start two servers: MJAPI (GraphQL backend, port 4000) and MJ Explorer (Angular UI, port 4200). From there everything is driven by metadata in your database.

Skip the ops — try MJC

Don't want to run the API and database yourself? MJC (MemberJunction Central) hosts MJ for you. Sign up, point it at your data, and get straight to building.

Prerequisites

  • Node.js 18+ — LTS recommended. Verify with node --version.
  • SQL Server 2019+ or PostgreSQL 14+ — local install, Docker, or a cloud instance.
  • Git — for cloning the sample project (optional).
  • An editor — VS Code recommended. The MJ VS Code extension gives you metadata browsing, entity editing, and inline CodeGen.

Install the CLI

npm install -g @memberjunction/cli

Verify the install:

mj --version
# memberjunction/cli@2.46.0

Scaffold a project

The CLI creates a complete MJ project — API, Explorer, configuration, and a starter migration — in one command:

mj init my-mj-app
cd my-mj-app

You'll get this layout:

my-mj-app/
├── packages/
│   ├── MJAPI/          # GraphQL backend (Apollo + Express)
│   ├── MJExplorer/     # Angular UI
│   └── GeneratedEntities/  # CodeGen output — typed entity classes
├── migrations/         # Flyway-managed SQL migrations
├── metadata/           # Declarative entity metadata (mj-sync managed)
└── mj.config.cjs       # Database + provider configuration

Configure your database

Edit mj.config.cjs with your database connection:

module.exports = {
    databaseSettings: {
        type: 'mssql',             // or 'postgres'
        host: 'localhost',
        port: 1433,
        database: 'MJ_Dev',
        user: process.env.MJ_DB_USER,
        password: process.env.MJ_DB_PASSWORD,
        defaultSchema: '__mj',
        connectionPool: { max: 10, min: 2 }
    },
    graphqlPort: 4000,
    explorerPort: 4200
};

Run the migrations to bootstrap the MJ schema and metadata tables:

mj migrate up
PostgreSQL or SQL Server, same code

MJ supports both database engines transparently. UUID casing differs between them — see the UUID Comparison guide if you'll be writing code that runs against both.

First run

Run CodeGen to generate entity classes, GraphQL schemas, and Angular forms from the bootstrap metadata:

mj codegen

Then start both servers (in two terminals, or use the workspace task):

# Terminal 1 — GraphQL API on :4000
npm run start:api

# Terminal 2 — Angular UI on :4200
npm run start:explorer

Explore in MJ Explorer

Open http://localhost:4200. You'll land on the MJ Explorer home screen. From the left nav you can browse:

  • Entities — every entity in the system, with CRUD forms auto-generated from metadata.
  • AI Agents — agents defined in metadata, with a chat interface and run history.
  • Actions — discoverable actions, with parameter forms and invocation history.
  • Dashboards — pre-built dashboards (AI Analytics, Knowledge Hub, Form Builder, etc.).
  • Queries — saved queries with parameter binding, executable from the UI or via GraphQL.

Create your first entity

The fastest way is to add a table to your database, then let MJ detect it. Add the SQL:

CREATE TABLE __mj.Project (
    ID UNIQUEIDENTIFIER NOT NULL DEFAULT NEWSEQUENTIALID(),
    Name NVARCHAR(200) NOT NULL,
    Status NVARCHAR(20) NOT NULL DEFAULT 'Active',
    StartDate DATE NULL,
    CONSTRAINT PK_Project PRIMARY KEY (ID)
);

Run CodeGen — MJ detects the new table, creates entity metadata, generates the TypeScript class, the GraphQL schema, the stored procedures, and an Angular form:

mj codegen

Reload MJ Explorer. The "Projects" entity now appears in the entity list with a working CRUD form, fully wired up to GraphQL.
