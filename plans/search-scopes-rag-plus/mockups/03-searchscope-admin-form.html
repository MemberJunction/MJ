About Search Scopes

A Search Scope is a named boundary that controls which providers, indexes, entities, and storage accounts participate in a search. Combined with a runtime Search Context, it enables multi-tenant, permission-aware retrieval.

When to use each tab

Definition — core metadata, time-windowing, RRF weights, re-ranker config.

Providers — which SearchProviders run inside this scope (plus optional per-provider query-rewrite templates).

External Indexes — vector stores (Pinecone/Qdrant/PGVector) and 3rd-party engines (Elasticsearch/Typesense/Azure AI Search).

Entities — scoped entity + FTS search, each with optional ExtraFilter and UserSearchString overrides.

Storage — storage accounts and optional folder-path restrictions.

Assigned Agents — agents that reference this scope via MJ: AI Agent Search Scopes rows.
