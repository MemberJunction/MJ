convertFilterToSQL detects a virtual field and emits dialect.JsonExtract(col, path) (already implemented for both platforms) with a type cast for numeric/date operators — instead of throwing "unknown field".
JSON-path predicates aren't index-backed. Fine at view scale; if a path gets hot, promote it to a persisted computed column + index (documented escape hatch).
