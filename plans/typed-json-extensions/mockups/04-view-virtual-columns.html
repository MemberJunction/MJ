What one row actually stores

Real columns + a single Enrichment JSON column. The grid never shows raw JSON — it projects paths into columns.
// Accounts row — "Acme Robotics"
{
  "Name": "Acme Robotics",
  "Status": "Active",
  "OwnerID": "…",
  "Enrichment": {
    "industry":  "Manufacturing",
    "employees": 2400,
    "arrTier":   "Tier 1",
    "hqRegion":  "NA-West"
  }
}

How a virtual column resolves

1 Schema on EntityField.ValueJSONSchema declares paths + types for Enrichment.
2 Column picker offers each path (e.g. $.arrTier) as a selectable column.
3 Display — AG Grid valueGetter extracts the path client-side. No server change for viewing.
4 Filter/sort — server emits dialect.JsonExtract(col, path) → JSON_VALUE (SQL Server) / ->> (Postgres).
Polymorphic constraint. When a JSON column's schema is chosen per-row (e.g. a Lists grid mixing List Types), virtual columns are only offered when the view is scoped to a single type. See List Detail grid.
