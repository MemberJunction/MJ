Calm Orb Ambient

Radical minimalism. The call is a beautiful, breathing presence — not a control panel. One thing at a time; everything secondary is one tap away.

The idea

The orb is the UI. By default you see the orb, the name, a single live caption, and exactly two controls (Mute, End). Captions / Type / Surface hide behind a single ••• that blooms a small sheet. In full screen, controls auto-fade after a few idle seconds — pure cinema.

Solves your pain points

  • Connecting never clips — the orb is the loader; it simply dims + three rising dots, identical footprint in any width.
  • Density — 2 visible controls instead of 9. The rest live in the bloom sheet.
  • Narrow/mobile — vertical, centered, thumb-reach dock; nothing competes for horizontal space.

Overlay vs full screen

Overlay = the whole story in 390px. Full screen = the same orb, larger, with an optional Conversation rail that slides in (timeline icon) and a fuller dock. No two-pane console — staying ambient is the point.

What becomes a widget @Input

<mj-realtime-overlay // host hides everything advanced: [Density]="'simple'" [ShowSurfacePanel]="false" [ShowActivityRail]="false" [ShowDevLinks]="false" [AutoHideControls]="true" [Compact]="isNarrow">

Command Console Structured

A premium, organized call surface. Clear zones, grouped controls, and a real two-pane console when you go full screen. Think Superhuman-meets-FaceTime for agents.

The idea

Tidy header (avatar + live dot + name + co-agent badge + state), a clean caption banner + transcript, and a fixed dock where controls are grouped into a segmented cluster (mic · captions · type) with End separated out. Advanced/dev settings live in a slide-in drawer — never on the call surface.

Solves your pain points

  • Connecting — a composed card: avatar + “Starting live session…” + a sleek progress bar. Same in any size.
  • Density — controls grouped (3-up cluster) instead of scattered; everything else in the gear drawer.
  • Surface panel only in full screen — the overlay shows a single “Expand” affordance; the whiteboard/browser/activity tabs appear only when there’s room.

Overlay vs full screen

Overlay = single conversation column + dock (no panel). Full screen = two panes: conversation+dock on the left, a gorgeous tabbed surface (Whiteboard / Browser / Activity) on the right, with the settings drawer on the gear.

What becomes a widget @Input

<mj-realtime-overlay [ControlLayout]="'cluster'" [ShowSurfacePanel]="isFullScreen" [ShowSettingsDrawer]="isFullScreen" [ShowDevLinks]="devMode" [Density]="isFullScreen ? 'pro' : 'standard'">

Floating Glass Spatial

Immersive + tactile. Frosted-glass islands float over the live surface; the agent presence hovers in space. Modern, playful-premium, and made for touch.

The idea

The surface (whiteboard / data / gradient) is the backdrop. Everything else — orb, transcript, captions, controls, activity — are floating glass cards with depth + blur. The control bar is one rounded glass pill; a ••• blooms extra chips. Full screen lets the islands breathe (and, conceptually, be repositioned).

Solves your pain points

  • Connecting — a single glass card + spinner ring centered; never clipped because it’s absolutely centered, size-independent.
  • Density — only 2 controls on the glass pill; the rest bloom as chips on demand.
  • Immersion — full screen shows the live surface behind, with captions floating near the orb and an Activity card top-left.

Overlay vs full screen

Overlay = orb up top, transcript flows beneath like glass messages, one floating control pill. Full screen = the surface fills the space; orb + captions + transcript + activity float as repositionable glass islands.

What becomes a widget @Input

<mj-realtime-overlay [Chrome]="'glass'" [SurfaceAsBackdrop]="isFullScreen" [ShowActivityRail]="isFullScreen" [FloatingControls]="true" [ShowCaptions]="true" [Compact]="isNarrow">

Shared: the new widget API

Across all three, the fix you asked for is the same — turn today’s hard-coded controls into declarative inputs so the overlay host can hide what doesn’t fit. The overlay sets a lean set; the full-screen / dedicated route opts into the rich set.
