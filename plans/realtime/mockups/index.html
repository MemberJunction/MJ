Real-Time Voice Agents

High-fidelity UX mockups for MemberJunction's new real-time voice agent experience — the live "call" overlay in Explorer's conversations, the delegation-to-Sage flow with on-screen result surfacing, interactive channels, the collapsed timeline session block, and the new admin tooling for sessions and channels.

End-User Experience
A
Live voice session
The focused call overlay over the conversation — agent identity, turn state, unified voice+text thread, channel strip and controls.
B
Delegation + result surfacing
Co-agent hands off to Sage with live step progress, then renders a result card inline with unmistakable "via Sage" provenance while it narrates.
C
Multi-channel · Whiteboard
Voice + an interactive whiteboard docked alongside the call, with agent-driven mutations highlighted distinctly from user edits.
C2
Live whiteboard channel · full board
The two-way whiteboard expanded inside the tabbed surface panel — drawing toolbar, ownership-legible items (you vs Sage), agent pop-in & presence cursor, "what Sage sees" perception popover, focus-board layout toggle, and a 4-beat storyboard.
D
Timeline · collapsed session
A normal chat thread with a single collapsed "Voice session" block — collapsed and expanded states with resume + Sage run links.
E
Session "behind the scenes"
End-user drill-in: overview, full transcript, delegated Sage runs with results, tool calls and usage/cost.
Admin Tooling — native MJ Explorer chrome

Rebuilt to match MJ's real admin patterns: the AI Dashboard's KPI tiles & charts, the AI Agent Run form's header / tabs / run-step tree / accordion details, and MJ data grids with status pills. These should feel lifted straight out of Explorer.

F
Realtime Voice analytics
Analytics dashboard in the AI Dashboard idiom — KPI tiles (active sessions, avg duration, delegated runs, janitor closes, cost), sessions-over-time & channel-usage charts, top target agents, recent sessions list.
G
AI Agent Sessions list
Dashboard list page — KPI tiles up top, then an MJ data grid of all AIAgentSessions (agent → target, user, status pills, channels, runs, tokens/cost, host instance) with filters, search and drill-in. Mirrors the AI Agent Runs list.
H
Session detail · entity form
A single session rendered as an MJ entity form — record-form-container toolbar, agent-run-style header + stat row, tabs, the delegated run tree (Co-Agent → Sage via ParentRunID with token/cost chips), lifecycle timeline, host/affinity facts, accordion details and transcript.
I
AI Agent Channels registry
The pluggable AIAgentChannel registry as an MJ list + entity-form edit — plugin classes (ClassFactory keys), transport-type pills, config schema, IsActive toggle and a form-container edit panel.

These are static, self-contained HTML/CSS mockups (Font Awesome via CDN, tiny vanilla JS for expand/toggle). Colors and chrome derive from MJ's dark-theme design tokens. Real transcript content from a live test session.
