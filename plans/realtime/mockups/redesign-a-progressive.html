How the console grows — two disclosure axes
Content-driven (within a session). Surfaces appear the instant the session produces their content — the first delegation slides the panel in with the Activity card, Sage's first board write births the Whiteboard tab, a finished delegation lands its artifact tab. Nothing is configuration-gated, and each arrival doubles as the progress indicator.
Capability-driven (across sessions, ratcheted). The composer, pickers, dev gear, full board toolbar and shortcut hints unlock as the user demonstrates depth. Milestones persist via UserInfoEngine mj.realtimeVoice.uxMilestones.v1 — once a control appears it never auto-hides. The gear's Simple / Standard / Pro density control is the manual escape hatch, writing the same setting.

The selected Console design, shown as it discloses itself: state 0 is pure audio — a breathing orb, three controls, no text, with a working Details peek at the (empty) Activity & Whiteboard panels; state 1 reveals the back-and-forth as text and unlocks typing; state 2 slides the surface panel in with the first delegation and docks the composer; state 3 births the Whiteboard + artifact tabs off Sage's first board write; and state 4 is the full Console from Redesign A — pickers, gear, shortcuts, stats. Scrub the 0 → 4 control above (or use ←/→ and 0–4) to watch it grow.
