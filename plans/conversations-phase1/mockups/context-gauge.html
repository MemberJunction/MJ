What: A pill-sized chip in the existing header-actions row — a conic ring + a single % number. Hover (or tap) reveals a popover with tokens used/limit, this-turn delta, run cost, and avg cost/message.

Why world-class: Zero chrome cost at rest — it sits next to Export/Share where users already look, and the ring reads like an Apple activity ring. Progressive disclosure: the 1% who care about exact tokens get the full breakdown without taxing the 99%.

Tradeoff: Hover-gated detail is undiscoverable on touch and easy to miss; the chip alone can't convey trend or per-message attribution.

What: A slim, always-visible progress bar pinned under the header (color-ramped, with tick marks per message contribution). Clicking it slides open a drawer that attributes tokens + cost per message and charts cumulative cost across turns.

Why world-class: Persistent ambient awareness with deep on-demand analytics — the bar is glanceable; the drawer turns "why is this conversation expensive?" into a one-click forensic view. Per-message attribution is genuinely actionable for power users and cost owners.

Tradeoff: Consumes vertical space every conversation, and the drawer competes with the artifact pane for screen real estate on narrow layouts.

Option C

Ambient Ring + Proactive Nudge

What: The composer itself is the gauge — a thin ring around the input fills and shifts green→amber→red as context grows, with a quiet token/cost line beneath. Crossing a threshold (~85%) triggers a one-time proactive nudge offering "Start fresh thread" or "Summarize & continue."

Why world-class: The signal lives exactly where the user's attention already is (the input they're about to type into), so it needs no hunting and no hover. It's the only option that's proactive — it intervenes before quality silently degrades and offers the fix in-flow, turning a hidden failure mode into a guided decision.

Tradeoff: A pulsing/coloring composer risks feeling alarmist or distracting; the nudge must be rate-limited and dismissible or it becomes nagware.

Data source (all three): Per-message tokens & cost are already captured on MJ: AI Agent Runs / MJ: AI Prompt Runs (input/output token counts + computed cost). The denominator — context-window limit — comes from the agent's resolved model (max-input-tokens on the selected MJ: AI Models row). The gauge aggregates run rows for the conversation; no new persisted state is required.
