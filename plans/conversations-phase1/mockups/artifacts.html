What: The artifact lives in the thread. Clicking it expands an inline editor with a left version-timeline rail and an Edit / Share / Remix toolbar. Saving an edit writes a new version (v3) authored by the user; the agent remains a collaborator on earlier versions.

Why world-class: Zero context switch — edit, branch history, and provenance sit exactly where the conversation produced them. The version rail makes "who changed what" legible, which is the hard part of co-editing with an agent.

Magic Link & privilege: The Share dialog mints a server-side read-only single-artifact Magic Link (RS256, expiring). The whole Magic-Link section is rendered only when the user holds "Can Publish Artifacts Publicly" — absent it, the section is omitted (not just disabled).

Tradeoff: An inline editor competes for vertical space inside the chat; complex artifacts feel cramped versus a dedicated surface.

Publish to web

Magic Link · read-only
Publish is shown only with Can Publish Artifacts Publicly; otherwise just Remix & internal share appear.

What: The ChatGPT/Gemini Canvas paradigm adapted to MJ's live runtime — chat on the left, the artifact rendered live on the right with direct editing, a Publish (Magic Link) button, and Remix. Agent edits and human edits both flow into the same canvas, each producing a version.

Why world-class: Tight conversational loop: you can talk or tinker, and the live preview reflects both instantly. Best surface for rich/interactive, data-aware "form-role" components that need room to breathe.

Magic Link & privilege: Publish mints the server-side read-only single-artifact Magic Link inline. The Publish button is conditionally rendered on "Can Publish Artifacts Publicly"; without it the toolbar shows only Remix + internal share.

Tradeoff: The split needs width — degrades on narrow screens, and the dual-pane layout is heavier to build/maintain than an inline card.

Share — "Q3 Revenue"

Magic Link · read-only
* The Share action that mints a public link only appears for users with Can Publish Artifacts Publicly. Others see internal-share only.

What: A dedicated artifact library (gallery) decoupled from any single thread. Each tile carries Edit / Versions / Share / Remix and a prominent Published (public link) vs Private badge. Filters pivot by ownership and visibility.

Why world-class: Treats sharing & remixing as first-class assets, not chat byproducts. Great for reuse at scale — find a teammate's published artifact, remix it into your own owned copy in a fresh conversation, and republish.

Magic Link & privilege: Share mints the server-side read-only single-artifact Magic Link (with a Revoke control). The link-minting Share action is gated on "Can Publish Artifacts Publicly" — hidden/disabled for users without it (see the "Forecast" tile). Remix clones into a new user-owned artifact in a new conversation.

Tradeoff: Moves artifacts away from their conversational context; users must hop to the hub, and provenance back to the originating chat needs deliberate surfacing.
