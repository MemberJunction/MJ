Standardized Agent Proxy

An MJ agent that is a thin proxy to a remote MJ-based / SaaS system (Skip, Betty, partner instances) — invoked in one step, passing a rich, MJ-aware context bag. Plan mode, skills, and local memory are delegated to the remote.

Delegated to remote: Plan mode, Skills, and Local Memory are turned off locally — the remote system owns reasoning, tool selection, and its own memory. MJ only assembles and ships the context bag.

What: Admin form to register a remote system as a first-class MJ agent — endpoint, auth, and per-field toggles for the context bag, with an explicit "delegated" callout.

Why world-class: Treats the rich MJ context bag as the product surface — admins see and govern exactly what crosses the boundary, not an opaque connector.

Tradeoff: Config depth can intimidate; needs sensible defaults so the common case is two fields, not twenty.

Each connector advertises its capabilities and the context it consumes — and notes that plan/skills/memory run remotely. Adding one drops a configured proxy agent into your picker.

What: A gallery / store of remote MJ & SaaS systems you can add as proxied agents in one click, each showing capabilities + exactly which context bag fields it shares.

Why world-class: An MCP-style marketplace, but MJ-native — connectors are self-describing about MJ entities/queries/notes, not generic tools.

Tradeoff: A curated store implies trust/vetting responsibility; partner/external entries need clear provenance to avoid blind installs.
