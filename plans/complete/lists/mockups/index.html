Lists Feature — Mockup Index

High-fidelity HTML mockups covering existing functionality and all envisioned enhancements across Phases 0–5. Each mockup is a standalone HTML file using shared MJ design tokens (styles.css).
For implementing agents and reviewers: these mockups define visual intent only — layout, hierarchy, copy, states, badge colors and meanings, drop-warning treatment, button placement. Production implementation must NOT copy the HTML/CSS from these files. Use MJ design tokens (_tokens.scss), @memberjunction/ng-ui-components, AG Grid, angular-split, and the patterns in CLAUDE.md. See plan §2.5 for the full rule.
