Four-layer stack with strict separation. Actions are a required parallel consumer (alongside Angular) so AI Agents and Workflows get the same surface as the UI. All flows that mutate go through the preview/commit contract.

Why Actions Are Required (Not Optional)

Every public capability that exists in the UI must also be exposed via an Action so that:
  • AI Agents can materialize from views, refresh, compose, and share lists without scraping the UI
  • Workflows can drive list ops as steps (e.g., "after invoice approved, add to Closed Won list")
  • Scheduled jobs can run nightly refreshes with audit trail
  • MCP tools get a first-class API for third-party orchestration
Actions are thin — each one delegates to a single ListOperations method, handling parameter mapping and result shaping only. They MUST honor the same confirmDrops contract: any Action that can produce drops requires an explicit input flag and the agent or workflow must supply it.

Drop-Warning Flow (All Layers)

[Angular | Action | MCP]  ─┐
                            ├──► ListsClient.computeDelta()  ──► Resolver  ──► ListOperations.computeDelta()
                            │                                                 ↓ returns ListDelta { add, remove, unchanged, deltaToken }
                            │
                            │   [If counts.remove > 0]
                            │      ┌── Angular: show <mj-list-delta-confirm> with required ack checkbox
                            │      ├── Action: caller MUST set confirmDrops=true input
                            │      └── MCP: agent must include confirmDrops in tool args
                            │
                            └──► ListsClient.applyDelta(deltaToken, confirmDrops)
                                    │
                                    ↓ Server validates:
                                       1. deltaToken HMAC is valid & not expired
                                       2. delta has not gone stale (re-compute on server)
                                       3. if counts.remove > 0 then confirmDrops MUST be true
                                       4. caller has Editor permission on target list
                                    ↓
                                  ApplyResult { added, removed, errors }
                                  + AuditLog entry (every mutation)
