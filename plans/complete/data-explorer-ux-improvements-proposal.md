# Data Explorer UX Improvement Proposal

## Executive Summary

This document proposes UX improvements for the MemberJunction Data Explorer application, leveraging the new metadata-driven dashboard system (`@memberjunction/ng-dashboard-viewer`). The goal is to transform the Data Explorer from a fixed-layout application into a flexible, personalized workspace where users can configure their ideal data browsing experience.

---

## Current State Analysis

### What Works Well
- **Entity browsing** with grid/cards/timeline views
- **Smart filter** with AI-powered natural language queries
- **View management** with full configuration persistence
- **Navigation panel** with favorites and recent items
- **Responsive design** for different screen sizes

### Pain Points Identified
1. **Fixed layout** - Users can't rearrange the sidebar, detail panel, or main content
2. **Home screen underutilized** - Application cards with minimal personalization
3. **No dashboard composition** - Can't save multi-entity layouts
4. **Limited quick access** - Only 5 recent items, no pinned dashboards
5. **Single entity focus** - No side-by-side entity comparison

## Technical Implementation

### New Dashboard Viewer Integration

The `@memberjunction/ng-dashboard-viewer` package provides:

1. **Golden Layout integration** - Drag-drop panel arrangement
2. **Pluggable panel renderers** - Via `@RegisterClass` pattern
3. **Config dialogs** - Per-panel-type configuration
4. **Event bubbling** - Navigation events bubble up to Explorer
5. **State persistence** - Save/load dashboard configurations

### Data Model Changes

The existing `Dashboard` entity with `UIConfigDetails` JSON field will store:
```json
{
  "version": 2,
  "layout": {
    "type": "row",
    "content": [
      { "type": "column", "content": [...] },
      { "type": "component", "componentState": { "panelId": "..." } }
    ]
  },
  "panels": [
    {
      "id": "panel-1",
      "partTypeId": "6C975185-297C-420D-BD59-F5402AF35399",
      "title": "Sales Pipeline",
      "icon": "fa-solid fa-chart-line",
      "config": { "viewId": "...", "maxRows": 100 }
    }
  ],
  "settings": {
    "autoSave": true,
    "theme": "light"
  }
}
```

### Migration Path

1. **Phase 1** - New dashboards use V2 config format
2. **Phase 2** - Migrate existing dashboard configs via CodeGen
3. **Phase 3** - Deprecate V1 format with fallback support

The key success factor is maintaining the existing entity browsing workflows while gradually introducing dashboard composition capabilities as an opt-in enhancement, not a replacement.
