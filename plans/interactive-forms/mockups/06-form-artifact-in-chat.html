The artifact viewer detects componentRole === 'form', auto-loads a Top-1 record from the declared entity, and renders the form against it. A record picker lets the user swap records. "Apply to my form" promotes this Component to an Override.

Recommendation (Claude's take): Option A. The whole point of the artifact viewer's intelligence is "we found a real record so you see a real form" — that's worth surfacing prominently. Hiding the picker makes the auto-binding feel magical, which is great until the auto-pick is wrong; then the user can't easily fix it. Discoverability > minimalism here.

Side note on entity inference: Both options assume the spec declares the entity. If the spec doesn't declare it (older Skip output), the artifact viewer falls back to fixture data with a small "Mock data" badge instead of the picker chip. We can mock that flow if you want — say the word.
