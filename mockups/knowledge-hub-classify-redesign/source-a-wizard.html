Set match and suggestion thresholds

Two cosine-similarity bands. Above MatchThreshold, the classifier auto-applies the existing tag. Between SuggestThreshold and MatchThreshold, the item routes to the Suggestions Inbox. Below SuggestThreshold, Auto-grow / Free-flow may invent a new tag (Hybrid won't).

SuggestThreshold 0.70
MatchThreshold 0.85
Cosine similarity. Higher = stricter. Default 0.85.
Must be lower than MatchThreshold. Default 0.05 below match.

What this means in practice

  • Score ≥ 0.85 → tag auto-applied · no human in the loop
  • 0.70 ≤ score < 0.85 → Suggestions Inbox · reason "Below threshold"
  • Score < 0.70 → in Hybrid: Suggestions; in AutoGrow: governance gates new-tag creation

Design notes — Option A

Wizard layout: each major decision (mode, scope, thresholds, budgets) gets a dedicated step with copy that teaches as it asks. Mode-cards on step 2 are the centerpiece — picking taxonomy posture is the most consequential choice and deserves big targets with explicit "best for" guidance. The bottom-of-page sneak-peek of the threshold step shows how the paired sliders are framed.

  • Strengths: first-time setup is gentle; "summary so far" panel always visible; mode cards make tradeoffs explicit; perfect for a sales-engineering demo.
  • Weaknesses: 4–6 clicks to change one knob; power users will want to skip steps; takes screen real estate.
  • Best for: initial source onboarding; non-technical operators.
  • Distinctive: mode-as-card with icons + "best for" copy; persistent summary aside; threshold-band ribbon visualization carried over from the Suggestions Inbox.
