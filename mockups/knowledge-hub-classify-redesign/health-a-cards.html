Design notes — Option A

Three summary cards, one per signal. Each card includes both the headline number and a "top 4" peek so the page tells the story without requiring drill-in. Threshold-tuning panel is right there — health is a knob-driven feature and the knobs should be visible. Run history at the bottom builds confidence that the job actually runs.

  • Strengths: calm and scannable; "top N" peek inside each card means you don't always need to drill; thresholds and history co-located; clear story (here are 3 problems, here's how each is detected).
  • Weaknesses: only 3 signals can fit; if we add a 4th the layout breaks; doesn't surface trends across runs.
  • Best for: weekly review by a data-governance owner.
  • Distinctive: 3-up card layout with embedded top-N preview; threshold knobs visible without leaving the page; run-history panel for trust.

B and C variants (described in index, not built): B (insights feed) would be a reverse-chronological stream of detected issues (Jira-Insights style) — better for tracking trends. C (tree heat-map) would color the taxonomy tree by health score — most beautiful, most expensive to build.
