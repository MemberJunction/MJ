v2 — three ways to layer reasoning onto the UI we already have

ℹ️ Different from v1. v1 reimagined the UX from a blank page. v2 is grounded in the real DuplicateDetectionResourceComponent — its Kanban (Pending/Approved/Rejected) and slide-in side-by-side comparison grid (records as columns, fields as rows, per-field survivor radios, dependency preview, merge-confirm). Each v2 pattern is a distinct philosophy for integrating the LLM layer into that existing structure.

All three keep the comparison grid + Kanban DNA and the existing merge mechanics; they differ in how invasively reasoning shows up. Same scenario as v1 (Accounts run #1842, the Acme 3-way match across Local/Salesforce/HubSpot). LLM elements use a distinct accent so they read as a separate layer over the vector metrics.

➕
A · Decorate

Augment in place

The exact comparison grid we ship today, plus three additions: a reasoning banner, per-match vec/LLM chips + survivor-recommendation badge, and per-field “LLM ✓” picks that preload SelectedColumnIndex.

Effort/risk: lowest — additive only. Fastest to ship.
🗂️
B · Reorganize

Recommendation board

Reshape the Kanban to triage by LLM verdict (Merge / Uncertain / Not-dup lanes), with a group-by toggle, confidence filter, “conflicts only”, bulk auto-merge, and a disagreements-need-a-human strip. Grid is the drill-in.

Effort/risk: medium — reworks the board, reuses the grid.
💬
C · Converse

Reasoning copilot

Grid on the left, a docked conversational reasoning panel on the right: verdict, explanation, the Agent's tool-investigation trace, applied memory notes, and “ask about this match”. Leans into Agent mode.

Effort/risk: higher — new panel; best showcases Agent mode.

A likely real build

  • Ship A first — it's purely additive on the working grid and matches the dual-provider Phase 1 (solo-prompt default).
  • Layer B's triage when run volumes grow and disagreement-first review pays off.
  • Reserve C for entities promoted to Agent mode, where the tool trace + memory + Q&A actually have something to show.
  • All three honor the spec: reasoning is annotation over the vector pass, survivor/field choices resolve to the existing FieldMap, and the merge path is unchanged.
