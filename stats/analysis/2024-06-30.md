# Analysis — 2024-06-30

Hand-written source code (TypeScript + JavaScript + HTML + CSS + Markdown) grew by roughly 25,000 lines this quarter, reaching 59,565. The bulk of that came from two fronts: the remote-change-detection work, which landed across more than two dozen PRs between March and June and added the `ChangeDetector` engine, external-change tracking views, and replay infrastructure to the server layer; and the communication-framework branch, which shipped across several merged PRs and introduced the messaging and notification plumbing visible in the new CSS and Angular HTML additions. A smaller but notable contributor was the `aq-agent-inside-action-entity` branch, which wired the Action entity into the AI agent system, and the `vectorize-records-when-looking-for-dupes` work that extended the vector/duplicate-detection pipeline.

The headline number for the quarter is SQL, which exploded from roughly 17K to 191K lines — a 10x increase. This is entirely explained by the installer script strategy adopted ahead of the v1.8 release: the team began committing cumulative baseline SQL scripts for every minor version milestone (v1.3 through v1.8), each a full `MJ_BASE_Structure` dump of 15K–28K lines alongside a matching `MJ_BASE_Dataset` seed file of 2K–3K lines, plus per-hop upgrade scripts covering both structure and data. By the June 30 snapshot there were six such baseline pairs in `SQL Scripts/install/`, plus seven upgrade-path scripts, accounting for essentially all of the 174K net SQL addition. This is hand-counted as hand-written (the classifier has no pattern for installer scripts), but it is largely mechanical — each baseline is a full schema dump regenerated by CodeGen and reformatted, not net-new business logic.

Generated TypeScript nearly quadrupled in absolute terms, adding 92K lines to land at 128K. That growth tracks directly with the major schema evolution across v1.3–v1.8: each CodeGen run regenerates `entity_subclasses.ts`, the full set of Angular form components in `core-entity-forms/src/lib/generated/`, and the GraphQL resolver stubs in `packages/MJServer/src/generated/generated.ts`. The communication framework and action-entity changes each introduced new database entities, and the change-detection work added update triggers across every updatable entity — all of which caused CodeGen to emit substantially larger output files. The net result is that generated code now accounts for 76% of all TypeScript and 69% of the combined source (TS + JS + HTML + CSS + MD) headline, up from a much lower fraction at the March baseline. The 547-file increase (928 to 1,475 tracked files) reflects both the new installer SQL scripts and the expanded CodeGen output surface.
