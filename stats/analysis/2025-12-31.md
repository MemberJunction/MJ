# Analysis — 2025-12-31

Hand-written source (TypeScript + JavaScript + HTML + CSS + Markdown) grew by +339,506 lines in Q4 2025, the second consecutive large quarter, bringing the total to 860,252. TypeScript led with +140,170 hand-written lines, CSS added +28,167 (a notable jump reflecting new component styling across several feature areas), and Markdown contributed another +151,180 — driven largely by the arrival of `packages/DBAutoDoc`, which shipped with roughly a dozen documentation files totaling hundreds of pages. The file count jumped from 3,541 to 5,214 (+1,673), again indicating new packages and test suites rather than incremental additions to existing ones.

The feature story for the quarter is split among four distinct effort areas. The `an-dev-4` branch (103 merge references) was by far the largest contributor, continuing the realtime and voice co-agent work from Q3 into a full ElevenLabs integration (`EL-BC`, 9 PR merges) that added realtime session UI, the voice overlay component, whiteboard channel refinements, and the `RealtimeClient` ElevenLabs driver. The `encryption-field-level` and `generic-credentials` / `comms-credentials` branches landed a new field-level encryption system and a credential-management entity layer, including a sizeable migration (`Add_Credential_Management_Entities.sql`, ~4,300 lines) and a parallel `Add_CredentialID_To_AI_Entities.sql` (~17,300 lines) that retrofitted AI entities with credential references. The `data-loading-optimization` branch (5 PR merges) shipped server-side RunView caching improvements, and `Component_Studio_Fixes` (5 PR merges) addressed the component linter and studio UI. Additionally, `DBAutoDoc` (a new package for AI-assisted database documentation) was committed this quarter with its full package structure, CLI binary, API, guardrails engine, and a committed test-run output that became the single largest new artifact in the repo.

The raw total line count more than doubled (1.6M → 3.74M), and the explanation is almost entirely in two generated artifact families. The JSON explosion (+1.16M generated lines) comes from two sources: `packages/DBAutoDoc/test-run/db-doc-state.json` was committed at ~40,000 lines as a persisted state snapshot of a tool run, and the `metadata/actions/` files — particularly `.biz-apps-social.json` and `.additional-core-actions.json` — grew substantially as the actions metadata layer was extended for new integration and agent-manager scenarios (~15,000 net inserted lines across those three files alone). The SQL surge (+515,632 total, +255,789 generated) is accounted for by DBAutoDoc test-run outputs: the `test-run/output/` subdirectory contributed dozens of committed `extended-props.sql` files ranging from 2,000 to 10,000 lines each, which the `repo-stats.mjs` classifier correctly marks as generated because they live under `packages/DBAutoDoc/test-run/`. The hand-written SQL growth (+259,843) reflects the credential and DateTime-field migrations noted above, plus the standard CodeGen-run SQL emitted alongside schema changes.

The generated-vs-hand split moved sharply in the generated direction this quarter: generated code now accounts for 43% of all tracked lines (up from a much smaller fraction), but that figure is almost entirely explained by the DBAutoDoc test-run SQL and JSON artifacts and the actions metadata expansion — none of which represent growing technical debt in the traditional sense. Stripping those artifacts out, the underlying source (TS+JS+HTML+CSS+MD) remains 86% hand-written, a ratio virtually unchanged from Q3. The TypeScript generated fraction actually shrank slightly (−6,705 generated TS lines) as CodeGen's entity class surface area stabilized relative to the hand-authored agent and realtime packages that dominate new growth.
