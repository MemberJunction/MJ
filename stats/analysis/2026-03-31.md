# Analysis — 2026-03-31

Hand-written source grew by a solid +783,944 lines in Q1 2026 — roughly 97% of the total line growth across TypeScript, HTML, CSS, and Markdown — continuing the sustained expansion of the previous two quarters but now spread across a broader front of feature areas rather than a single dominant track. TypeScript led in absolute terms (+289,397 hand-written lines), followed by HTML (+229,882) and Markdown (+149,342). The file count jumped from 5,214 to 7,772, adding 2,558 tracked files in a single quarter, the largest file-count increase in the repo's recorded history and a strong signal that net-new package creation (rather than growth within existing packages) was the predominant mode of work.

The biggest functional investments were in infrastructure, AI, and the v5 migration era. The `MJ_v5_Migration`, `MJ_v4`, and related `Fixing_v3_Issues` branches landed the multi-version baseline consolidation and schema cleanup work that underpins the v5.x release series (versions 5.13 through 5.21 were tagged during the quarter, each a mj-upgrade PR). The `feature/runtime-schema-update` track — open since late Q4 and merged in Q1 — shipped live schema-diff detection so CodeGen can apply incremental changes without a full regeneration pass. On the AI side, `JF_prompt_runner` extended the prompt runner infrastructure, `arie/memory-manager-agent` added a memory management sub-agent, `claude/elevenlabs-agent-integration` wired ElevenLabs realtime voice into the agent session stack, and `multi-modal-agent-output` added image-generation output paths to the agent framework. The `claude/slack-teams-integration` and `salesforce-integration-connector` PRs — along with the `actions-integrations-unification` track and a wave of `feat/` and `CB-composable-queries` branches — expanded the integrations layer substantially, which explains much of the HTML growth (new Angular component templates for connectors and chat UI). The `ian-basesingleton-migration` PR continued the systematic replacement of manual `static _instance` patterns with `BaseSingleton<T>`, touching dozens of classes across the codebase. API key management (`api-keys-for-mjapi`, `api-key-design`, `api-keys-passed-to-subagents`) and MCP OAuth (`601-mcp-oauth`, `001-mcp-oauth-dcr`) each arrived as substantial new packages. The Markdown surge (+149,342 lines) is largely the `JF_readme` and `open-issues-and-doc-improvements` tracks plus guide documents committed alongside new packages — all classified hand-written since no Markdown generation pattern exists.

The generated-SQL explosion (+918,941 lines) is the defining feature of the generated-vs-hand split this quarter and needs to be read carefully. Three overlapping forces drove it: first, the `MJ_v3_Baseline_Migration_Optimization` work produced consolidated baseline SQL for v3 and v4 schema state, substantially padding the `migrations/v*/B*__*.sql` baseline family. Second, the `migrations-pg` top-level directory — a PostgreSQL mirror of the SQL Server migrations produced by `pg-migrate` — appeared on 2026-03-01 and accounts for a full parallel copy of migration content, all of which is classified generated. Third, regular CodeGen runs during the version bumps (5.13–5.21) each appended `CodeGen_Run_*.sql` files with stored-procedure, view, and entity-field SQL, the normal per-release cadence but amplified by the schema surface added across all the new packages. The net result is that generated code now accounts for 49% of SQL lines and 45% of all tracked lines, but only 9% of the source-code headline (TS+JS+HTML+CSS+MD), unchanged from Q4 — the ratio in the lines that matter most for product complexity stayed flat even as the tooling-output volume tripled.
