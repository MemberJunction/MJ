# Analysis — 2024-03-31

This snapshot captures the initial import of the MemberJunction codebase and the first several months of active development through Q1 2024, growing from an essentially empty repo (2 files) to 928 tracked files and 127,721 lines of code. The headline figure is 34,294 lines of hand-written source (TypeScript, JavaScript, HTML, CSS, and Markdown), with hand-written TypeScript alone accounting for 27,495 of those — a strong signal that the core framework, providers, and Angular components arrived largely complete rather than being assembled incrementally over this period.

The biggest feature work visible in the git log is the build-out of the Angular Explorer UI: a generic browser/grid component, tree-view navigation, resource-based routing, CRUD operations wired to the grid, file storage (with AWS and Google drivers), a record duplicate UI, and an admin console covering users, roles, and entity permissions. Alongside that, the AI package layer was established — Anthropic, Gemini, and Mistral wrappers shipped, and a short-lived `mjaiengine` package was created then removed and consolidated — and vector packages landed toward the end of the window. Several CodeGen runs are explicitly called out in commit messages, each regenerating entity classes, Angular forms, stored procedures, and related SQL objects after schema changes such as adding `Sequence` to `EntityRelationship`, introducing CHECK constraints that feed `EntityFieldValues`, and enabling delete capabilities on additional entities.

The generated-vs-hand split reflects that mix: 57% of all TypeScript is generated (CodeGen entity subclasses, GraphQL resolvers, Angular form components) and 87% of JSON is generated (primarily `package-lock.json` and manifest files across the monorepo's many packages). SQL and CSS are entirely hand-written at this snapshot — migrations and baseline schema objects were still authored by hand, and no pg-migrate conversion or mj-sync metadata migrations had landed yet. The 7,790-line GraphQL schema (two files, counted under "Other") is also hand-written, representing the Apollo schema definitions that back the data provider layer. In total, generated code accounts for 51% of all tracked lines, which is consistent with a mature metadata-driven framework arriving in largely finished form at initial import.
