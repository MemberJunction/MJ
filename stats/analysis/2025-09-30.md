# Analysis — 2025-09-30

Hand-written source added 130,196 lines in Q3 2025 (TypeScript +58,228, Markdown +63,978, CSS +11,215, HTML +20,413, JavaScript +29,362), bringing the total hand-written source count to 521,292 lines across 3,541 tracked files — a steady quarter of feature work rather than a structural explosion like Q2. Total file count grew by 715, again indicating new-package and new-file creation rather than only growth inside existing files.

The quarter was dominated by the `an-dev-agents-2` branch, which produced 121 individual merges — more than three times any other branch — continuing the agent-framework buildout from Q2. That work extended the `BaseAgent` runtime with payload read/write paths (PR #1404), run-chaining for AI agent sessions (v2.63.x migration), failover prompt strategies (v2.62.x), and flow-agent enhancements including array-indexing support in `ActionInputMapping`. In parallel, the `JF_Skip` and `JF_Dev_2` branches contributed a new component-linting system for RunQuery validation, fixes to the component registry cache, React component system improvements, and the initial v3 conversations/artifacts/tasks schema (a 744-line `schema.sql` plus migration skeleton that will underpin a forthcoming redesign of the conversations surface). The `Ethan-BC` branch landed external-API integrations: a Box SDK v10 storage driver, a Gamma API integration for presentation generation, a Perplexity search action, and a centralized configuration system for the MJStorage driver family — all shipped as discrete `Actions` packages with associated metadata. The `Component_Linter_Missing_Query_Rule` and `ComponentRegistry_Cache_Fix` PRs (#1398, #1410) addressed correctness issues in the component studio introduced during the prior quarter.

The JavaScript line count jumped from 3,537 to 32,899 (+29,362), which looks dramatic but has a straightforward explanation: a large library of build-less React component files was committed under `metadata/components/code/` as plain `.js` source — accounts-by-industry charts, AI model analytics panels, activity feeds, aging charts, and dozens more. These are hand-authored (no generation pattern matches them), so the entire delta lands in the hand-written bucket. The 245K SQL increase is similarly explainable: 115 migration files under `migrations/v2/` were added across the quarter (v2.62.x through v2.104.x), carrying schema work for agent run-chaining, collection-artifact join tables, new AI model entries (Grok 4 fast models), action unique-constraint changes, and the v3 conversations draft scripts. The Markdown growth of +63,978 lines comes from a combination of changeset entries under `.changeset/`, architectural proposal documents in `metadata/components/` and `v3_conversations/`, and a steady expansion of CLAUDE.md sub-docs and developer guides committed alongside the features they describe.

The generated-vs-hand split moved further toward hand-written: generated code now accounts for 16% of source lines and 10% of all tracked lines, down from 27%/14% at the prior snapshot. The counterintuitive drop in generated TypeScript (−22,278 lines, from 113K to 91K) reflects a CodeGen restructuring that consolidated or removed previously emitted entity subclasses and form boilerplate rather than any reduction in schema coverage — the hand-written TypeScript grew by +58,228 in the same period, confirming that the overall entity and UI surface expanded. Generated JSON rose by +15,638 lines, consistent with new metadata sync output and lockfile updates from the version-pinning work that kept `@memberjunction/core` and `@memberjunction/global` fixed at v2.100.3 while bumping all other packages through v2.101.0 and v2.102.0.
