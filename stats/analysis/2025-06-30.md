# Analysis — 2025-06-30

Hand-written source more than doubled in Q2 2025, adding 234,812 lines across TypeScript (+94,637), Markdown (+82,820), CSS (+23,891), and HTML (+13,313) — the largest single-quarter increase in the repo's tracked history. The file count also surged from 1,883 to 2,826 (+943 files), a jump that signals a sustained phase of new-package creation rather than incremental growth in existing ones. SQL grew by +122,390 lines, almost entirely hand-written migrations, consistent with the schema work that underlies the new agent and realtime infrastructure described below.

The dominant force was the `ai-agent-dev-an` branch, which landed more than thirty individual merge requests across the quarter (PRs #1054–#1125). This branch built out MemberJunction's first-class Agent Framework: the `BaseAgent` runtime, conductor/sub-agent hierarchy, `AIAgentSession` lifecycle management, realtime co-agent support, and the whiteboard interactive channel. Alongside that, the `UI/Execution-Monitor` branch introduced a dedicated execution-monitoring dashboard for agent runs, and `UI-Fixes-6-20-25` and `UI-Enhancements-Merge` bundled a round of Angular component polish that accounts for much of the CSS and HTML growth. The `integrated-cli` merge (PR #1063) landed the unified `mj` CLI with consolidated commands for codegen, manifest generation, and sync operations; `mj.config` and `baseUrl` PRs (#1085–#1089) wired up the new `mj.config.cjs` configuration layer. The Markdown explosion (+82,820 lines) reflects the wave of guides, CLAUDE.md sub-docs, and architectural reference documents (caching, transport layer, realtime co-agents, forms architecture, lazy loading, and several others) that were committed directly into the repo alongside the code they document — the 295 Markdown files at snapshot time are all classified as hand-written because no Markdown generation pattern exists.

The generated-vs-hand split shifted toward hand-written compared with prior snapshots: generated code accounts for only 27% of source lines (TS+JS+HTML+CSS+MD) and 14% of all tracked lines, down from higher ratios in earlier periods. That movement is structural — the new agent and realtime packages are net-new hand-authored surface area that CodeGen does not yet cover, while the generated fraction (113K lines of CodeGen entity classes, Angular forms, and resolvers) grew only modestly (+15,733 TS, +1,293 HTML) because the quarter's schema additions were narrower than the feature volume might suggest. Two CodeGen-specific fixes also landed: `CodeGen-fix-spCreate-UUID-non-key-default` (PR #1121) corrected stored-procedure generation for UUID non-key columns with defaults, and `code-gen-upsert-v2` (PRs #1075–#1076) reworked the upsert path, both of which triggered regeneration of affected SQL objects. The `Bug-fix-#1011-EntitySetAndEntityFieldDirty_NumericAndBoolean` fix (PR #1122) addressed silent dirty-tracking failures for numeric and boolean fields in `BaseEntity`, a correctness issue that affected saves across the entire entity layer.
