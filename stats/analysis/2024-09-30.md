# Analysis — 2024-09-30

Hand-written source code (TypeScript + JavaScript + HTML + CSS + Markdown) grew by roughly 17,500 lines this quarter, reaching 77,139. The TypeScript component of that was about 6,500 new hand-written lines, with the remaining growth spread across SQL (+62K) and Markdown (+11K). Feature-wise, the quarter was broad rather than deep: the `communication-changes` branch wired MS Graph as a communication provider (new migration and package work in late September), the `scheduled-actions` and `vector-packages` branches continued maturing those subsystems with error handling and worker improvements, the `recommendation-engine` branch (`update-rec-engine`) added further refinements, and `List-functionality` restored entity list creation. CodeGen itself received two notable improvements — support for virtual entities (`f5f25d08`) and handling of special characters and array types in the schema reader (`codegen-special-characters`, `codegen_arrays`). TypeDoc configuration was also added to every package in a single sweep PR (#528). SQL growth (+62K lines) is again dominated by cumulative baseline and migration scripts rather than net-new business logic.

The headline number to explain this quarter is the 37,000-line shrink in generated TypeScript, which caused total TS to fall from 167K to 137K even as the file count rose by 118. The root cause was a consolidation of generated entity subclasses: the `packages/GeneratedEntities/src/generated/entity_subclasses.ts` file, which held roughly 48,000 raw lines at the June snapshot, was emptied down to a 10-line stub. Its content migrated to `packages/MJCoreEntities/src/generated/entity_subclasses.ts`, which grew from ~21K to ~34K lines — absorbing roughly 13K of those 48K. The remaining ~35K lines represent genuine deduplication: entities that had been maintained in two packages are now authoritative only in MJCoreEntities. The parallel Zod schema integration (PR #438 / #483) also changed the internal structure of the generated file — field descriptions that were previously emitted as JSDoc block comments (counted by cloc as comment lines) are now inlined as Zod `.describe()` string arguments (counted as code), which partly shuffled the blank/comment/code breakdown without changing net information content.

The generated-vs-hand split shifted noticeably as a result: generated TypeScript fell from 76% to 67% of all TS lines, and generated code as a share of the combined source (TS + JS + HTML + CSS + MD) fell from 69% to 56%. This is a healthy direction — the reduction reflects consolidation of redundant output, not a retreat from generated code. Hand-written TypeScript's share rising from 24% to 33% is similarly an artifact of the denominator shrinking rather than a surge in hand-authored logic. Generated HTML grew slightly (+1K lines, now 68% of all HTML) as CodeGen emitted new form components for entities added in migrations this quarter.
