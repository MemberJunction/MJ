# Repository Stats (`/stats`)

This folder tracks lines-of-code snapshots over time. It is fully generated by
[repo-stats.mjs](repo-stats.mjs) — **do not hand-edit** `README.md`, `data.csv`, or anything
in `reports/`; re-run the script instead.

## How to record a snapshot

```bash
node stats/repo-stats.mjs               # snapshot the current working tree (dated today)
node stats/repo-stats.mjs <commit>      # backfill a historical commit (dated by commit date)
node stats/repo-stats.mjs <c1> <c2> …   # backfill several commits in one run
```

Requires [cloc](https://github.com/AlDanial/cloc) on the PATH (`brew install cloc`).
No npm dependencies — the script is plain Node, run it from anywhere in the repo.

Each run:

1. Counts git-tracked files only (`cloc --vcs=git`, or `cloc --git <sha>` for historical
   commits — no checkout needed), so `node_modules`, `dist`, and gitignored paths are
   excluded automatically.
2. Upserts a row into `data.csv` keyed by date (re-running on the same day replaces that
   day's row — safe and idempotent).
3. Writes a full per-language report to `reports/YYYY-MM-DD.md`.
4. Regenerates `README.md` (history table + mermaid trend charts, which render on GitHub).

## Backfilling historical points

Find the commit at a date boundary, then pass it to the script:

```bash
sha=$(git rev-list -1 --before="2026-01-01 00:00" HEAD)
node stats/repo-stats.mjs "$sha"
```

The existing history is quarterly. Keep new backfill points on quarter boundaries unless
there's a reason not to; ad-hoc current-tree snapshots at any time are fine.

## Conventions

- **Language buckets** are defined in `LANGUAGE_GROUPS` at the top of the script; anything
  unlisted rolls into "Other". "Source" (the headline number) = TypeScript + JavaScript +
  HTML + CSS + Markdown — see `SOURCE_GROUPS`. SQL and JSON are tracked but reported
  separately because they're dominated by generated migrations and declarative metadata.
- **Adding a new bucket** means a new column in `data.csv` — old rows will have empty cells,
  so either re-run the backfill for all commits in `data.csv` or accept blanks.
- Snapshots are cheap (~20s on the current repo) and the script touches nothing outside
  this folder, so it's always safe to run.
