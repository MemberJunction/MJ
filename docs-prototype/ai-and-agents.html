AI & Agents

15+ providers behind one interface. Agents that can call your data. MCP and A2A for interop.

The unified AI engine

Every AI capability in MJ — chat, embeddings, image generation, audio, video, reranking — is exposed through provider-agnostic interfaces. Switching from OpenAI to Anthropic to a self-hosted Ollama model is a config change, not a code change.

const ai = new AIEngine();
const result = await ai.ChatCompletion({
  model: 'gpt-4o',          // or 'claude-sonnet-4-5', 'gemini-pro', 'llama-3', ...
  messages: [{ role: 'user', content: 'Summarize this quarter\'s metrics' }],
  data: await rv.RunView({ EntityName: 'Quarterly Reports', ResultType: 'simple' })
});

Protocols: MCP & A2A

MCP Server

Expose every MJ entity (CRUD) and every AI agent (execution) as a tool to any MCP-compatible client — Claude Desktop, Cursor, Zed, custom builds.

@memberjunction/ai-mcp-server →

A2A Server

Google's Agent-to-Agent protocol for cross-platform agent interop. Lets MJ agents collaborate with agents built on different platforms.

@memberjunction/a2aserver →

Prompt Engine

Hierarchical prompt templates with system placeholders, model selection, effort-level control, parallel execution, and execution tracking. Every prompt run is recorded as an entity for audit, replay, and analysis.

@memberjunction/ai-prompts →
