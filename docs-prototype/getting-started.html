Getting Started

Install MJ, get the API and UI running, and write your first entity.

Prerequisites

  • Node.js 20+ and npm 9+
  • SQL Server 2019+ or Azure SQL (PostgreSQL is supported but not the primary path yet)
  • Angular CLI 21+ for working on the UI
  • An auth provider: Auth0, Azure AD (MSAL), or Okta
  • Optional: Docker, if you want the workbench dev environment

1. Clone and install

git clone https://github.com/MemberJunction/MJ.git
cd MJ
npm install

This is an npm workspace monorepo with 175+ packages — install runs at the root.

2. Configure

cp install.config.json.example install.config.json
# Edit install.config.json with:
#   - Your SQL Server connection string
#   - Your auth provider settings (Auth0 / MSAL / Okta)
#   - Your default schema name

For a deeper config tour, see @memberjunction/config and the root mj.config.cjs.

3. Initialize the database

node InstallMJ.js

This runs Flyway migrations from migrations/v5/, creates the MJ system schema, and seeds reference data. See Migrations for the full migration story.

4. Run it

npm run start:api        # GraphQL API on port 4000
npm run start:explorer   # Angular UI on port 4200

Visit http://localhost:4200, sign in via your configured auth provider, and you're looking at MJ Explorer.

5. Your first entity

The "hello world" of MJ is a database table. Create one in your default schema (or use an existing app schema), then run CodeGen:

npm run codegen

CodeGen detects the new table, generates a typed entity class, server-side resolvers, and an Angular form. The full pipeline is documented in the CodeGenLib README.

From your TypeScript code, the new entity is available immediately:

import { Metadata } from '@memberjunction/core';

const md = new Metadata();
const customer = await md.GetEntityObject<CustomerEntity>('Customers');
await customer.Load(customerId);
customer.Status = 'Active';
await customer.Save(); // Validation, audit, downstream actions all automatic
