Contribute

MemberJunction is open source. Contributions are welcome at every level — bug fixes, docs improvements, new providers, new packages.

  1. Read CLAUDE.md — coding standards, naming conventions, anti-patterns, and architectural guidelines. This is the developer handbook and your PR reviewer's reference.
  2. Open an issue before starting non-trivial work. We'll confirm direction so your effort isn't wasted.
  3. Branch from next with a descriptive name (e.g., fix/cache-invalidation-edge-case).
  4. Run tests in the affected packages: cd packages/PackageName && npm run test. Tests must pass before review.
  5. Submit a PR with a clear summary and a test plan.

Reporting a security issue

Please don't open a public issue for security vulnerabilities. Email support@memberjunction.org with details, and we'll respond promptly with coordination.
